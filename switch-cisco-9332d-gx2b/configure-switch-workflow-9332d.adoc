= Configure switch workflow - Cisco Nexus 9332D-GX2B
:icons: font
:imagesdir: ../media/

[.lead]
You configure Cisco 9332D-GX2B switches as part of a new AFX system installation, a switch replacement, or a reset to factory default settings. 

.Before you begin

* If you are installing and setting up switches as part of new AFX A1K or AFX A2K system installation and setup, please see the link:configure-switch-overview-9332d-cluster.html[Install and setup switches for AFX system workflow].

* You should have been directed here as part of one of the following procedures:

** Installation and setup of a new AFX A1K or AFX A2K system.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-1.png[One]link:setup-switch-9332d-cluster.html[Complete the initial switch setup]
[role="quick-margin-para"]
After you power on the switches, they begin to boot and enter the configuration wizard. You are prompted to provide answers to a series of questions to perform the initial (startup) configuration of the switches.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-2.png[Two]link:install-nxos-overview-9332d-cluster.html[Prepare to install NX-OS and RCF]
[role="quick-margin-para"]
Prepare to install NX-OS and RCF on the switches.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-3.png[Three]link:configure-software-overview-9332d-cluster.html[Install the software]
[role="quick-margin-para"]
Install the current NX-OS on the switches.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-4.png[Four]link:configure-software-overview-9332d-cluster.html[Install the RCF]
[role="quick-margin-para"]
Install the version of RCF on the switches that supports your AFX A1K or AFX A2K system.

//This step is embedded in install and upgrade RCF procedures:
//.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-5.png[Five]link:change-port-speed-9332d-cluster.html[Change port speed configuration (optional)]
//[role="quick-margin-para"]
//For AFX A2K systems with switches running RCF 10.1 or later, you can change the number of switch ports in breakout mode (4x100GbE) and 400GbE mode when you need to accommodate a different port speed configuration for your system.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-5.png[Five]link:configure-software-overview-9332d-cluster.html[Verify SSH and SSH keys are enabled]
[role="quick-margin-para"]
Verify SSH and SSH keys are enabled. SSH is required for the Cluster Switch Health Monitor (CSHM) and log collection features to operate.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-6.png[Six]link:../switch-cshm/cshm-overview.html[Configure cluster switch health monitoring]
[role="quick-margin-para"]
Configure the cluster switch health monitoring (CSHM) to ensure the operational health of your switches and for the collection of switch logs for debugging and troubleshooting.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-7.png[Seven]link:smart-call-9332d-cluster.html[Configure Smart Call Home (optional)]
[role="quick-margin-para"]
Configuring Smart Call Home on your switches is optional. It is used to monitor the hardware and software components on your network. 

.What's next?

If you configured your switch as part of a new AFX A1K or AFX A2K system installation, you must link:complete-cluster-setup.html[Complete the AFX system setup].

If you configured your switch as part of a switch replacement or you reset the switch to factory default settings, your switch should be operating normally.
