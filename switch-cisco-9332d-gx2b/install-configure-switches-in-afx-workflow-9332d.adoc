= Install and configure switches in an AFX system workflow - Cisco Nexus 9332D-GX2B

:icons: font
:imagesdir: ../media/

[.lead]
To install and configure your Cisco Nexus 9332D-GX2B switches as part of your new AFX A1K or AFX A2K system, you use a combination of documentation from this switch site and the AFX system site. The documentation directs you with links as needed.

=== Install switches
To install your switches, you must prepare for the installation and then install the switches (along with the AFX system hardware), which includes powering on and cabling the switches.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-1.png[One]link:install-prepare-9332d-cluster.html[Prepare for install]
[role="quick-margin-para"]
Prepare to install and setup the switches by reviewing the Cisco and ONTAP systems documentation, preparing your site, and understanding the switch port usage.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-2.png[Two]link:install-hardware.html[Install the switches]
[role="quick-margin-para"]
You are directed to the AFX A1K or AFX A2K system installation and setup instructions to install the switches (along with the AFX system hardware), cabling the switches, and then power on the switches. You are then directed back to this workflow to configure your switches.

=== Configure switches
To configure your switches, you complete the initial setup, prepare to install NX-OS and RCF, install NX-OS, install RCF, verify SSH is enabled, configure cluster switch health monitoring (CSHM), and configure Smart Call Home (optional).

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-1.png[One]link:setup-switch-9332d-cluster.html[Complete the initial switch setup]
[role="quick-margin-para"]
After you power on the switches, they begin to boot and enter the configuration wizard. You are prompted to provide answers to a series of questions to perform the initial (startup) configuration of the switches.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-2.png[Two]link:install-nxos-overview-9332d-cluster.html[Prepare to install NX-OS and RCF]
[role="quick-margin-para"]
Prepare to install NX-OS and RCF on the switches.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-3.png[Three]link:install-nxos-software-9332d-cluster-test.html[Install or upgrade the NX-OS]
[role="quick-margin-para"]
Install the current version of NX-OS on the switches that supports your AFX A1K or AFX A2K system.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-4.png[Four]link:install-upgrade-rcf-overview-cluster.html[Install the RCF]
[role="quick-margin-para"]
Install the version of RCF on the switches that supports your AFX A1K or AFX A2K system.

//This step is embedded in install and upgrade RCF procedures:
//.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-5.png[Five]link:change-port-speed-9332d-cluster.html[Change port speed configuration (optional)]
//[role="quick-margin-para"]
//For AFX A2K systems with switches running RCF 10.1 or later, you can change the number of switch ports in breakout mode (4x100GbE) and 400GbE mode when you need to accommodate a different port speed configuration for your system.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-5.png[Five]link:configure-ssh-keys.html[Verify SSH and SSH keys are enabled]
[role="quick-margin-para"]
Verify SSH and SSH keys are enabled. SSH is required for the Cluster Switch Health Monitor (CSHM) and log collection features to operate.
//SSH enablement is part of the switch initial setup in Configuration Wizard, so it should be enabled.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-6.png[Six]link:../switch-cshm/cshm-overview.html[Configure cluster switch health monitoring]
[role="quick-margin-para"]
Configure the cluster switch health monitoring (CSHM) to ensure the operational health of your switches and for the collection of switch logs for debugging and troubleshooting.


.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-7.png[Seven]link:smart-call-9332d-cluster.html[Configure Smart Call Home (optional)]
[role="quick-margin-para"]
Configuring Smart Call Home on your switches is optional. It is used to monitor the hardware and software components on your network. However, using cluster switch health monitoring (CSHM) is recommended instead.


=== Complete AFX system setup
To complete the AFX system setup, you complete the power up of AFX system components and system setup procedures, and then verify switch connectivity.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-1.png[One]link:complete-cluster-setup.html[Complete the AFX system setup]
[role="quick-margin-para"]
After you've completed the configuration of your switches, you complete the setup of your AFX A1K or AFX A2K system by powering on the storage shelves and controllers, completing the AFX A1K or AFX A2K system setup procedures, and then verifying the switches are properly connected to each other and to your AFX system.
